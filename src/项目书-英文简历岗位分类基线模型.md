# 基于英文简历的岗位类别分类项目书

## 一、项目名称

基于英文简历文本的岗位类别自动分类系统

## 二、项目背景

在招聘场景中，企业每天会收到大量求职者简历。人工筛选简历不仅耗时，而且容易受到主观因素影响。若能够根据简历内容自动判断候选人所属岗位方向，例如人力资源、信息技术、教师、销售、金融、工程等类别，就可以辅助 HR 快速完成简历初筛、岗位归类和后续分发。

本项目计划先使用原版英文简历数据集构建基线模型，验证简历文本分类任务的可行性。基线阶段采用两条路线：第一条是 `TF-IDF + XGBoost`，用于建立结构清晰、可解释性较强的机器学习基线；第二条是 `FastText supervised`，用于建立轻量级深度学习文本分类基线。

后续可在 XGBoost 和 FastText 的结果基础上升级为 BERT 系列模型。由于简历文本普遍较长，普通 BERT 直接截断会丢失后半部分经历、技能和教育信息，因此本项目将采用 `Hierarchical BERT` 作为后续深度学习主线方案，以提升模型对长简历文本的整体理解能力，并形成“机器学习 baseline -> 轻量深度学习 baseline -> Hierarchical BERT 长文本模型”的完整模型演进路线。

## 三、数据集介绍

### 1. 数据集来源

该数据集来自 LiveCareer 网站中的简历示例，由数据集作者通过网页爬虫采集整理。

数据集中包含 2400 多份英文简历，每份简历被标注为一个岗位类别。原始数据同时包含文本格式和 HTML 格式，页面介绍中也说明 PDF 文件按照类别文件夹进行存储，每个 PDF 文件名与 CSV 中的 `ID` 对应。

### 2. 数据集用途

该数据集用于将给定简历自动分类到指定岗位类别中，是一个典型的多分类文本分类任务。

输入：

```text
英文简历文本
```

输出：

```text
岗位类别
```

例如：

```text
输入：包含招聘、员工关系、薪酬福利、培训等内容的英文简历
输出：HR
```

```text
输入：包含 Python、SQL、Linux、系统开发、数据库等内容的英文简历
输出：INFORMATION-TECHNOLOGY
```

## 八、基线模型建设目标

本阶段目标不是直接追求最优模型，而是先构建一个可运行、可评估、可复现的 baseline。

主要目标包括：

1. 完成原始英文简历数据读取和清洗。
2. 完成岗位类别标签编码。
3. 使用 TF-IDF 提取文本特征。
4. 训练 XGBoost 和 FastText 两类 baseline 模型。
5. 输出准确率、宏平均 F1、加权 F1、分类报告和混淆矩阵。
6. 保存 XGBoost 模型、FastText 模型、TF-IDF 向量化器和标签映射文件。
7. 为后续 BERT 模型升级提供对比基准。

## 九、技术路线

### 1. 数据读取

读取原始 CSV：

```python
import pandas as pd

df = pd.read_csv("Resume.csv")
```

使用字段：

```python
texts = df["Resume_str"]
labels = df["Category"]
```

### 2. 数据清洗

基线阶段建议进行轻量清洗：

- 去除首尾空格。
- 合并多余空白字符。
- 去除异常过短文本。
- 保留英文大小写或统一小写。
- 暂不删除专业词汇。

示例：

```python
import re

def clean_text(text):
    text = str(text)
    text = re.sub(r"\s+", " ", text)
    return text.strip()

df["clean_text"] = df["Resume_str"].apply(clean_text)
```

### 3. 标签编码

将岗位类别转换为数字标签：

```python
from sklearn.preprocessing import LabelEncoder

label_encoder = LabelEncoder()
df["label"] = label_encoder.fit_transform(df["Category"])
```

保存 `label_encoder`，方便预测结果还原为类别名称。

### 4. 数据集划分

由于类别不均衡，建议使用分层抽样：

```python
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    df["clean_text"],
    df["label"],
    test_size=0.2,
    random_state=42,
    stratify=df["label"]
)
```

### 5. XGBoost 特征工程

XGBoost 不能直接处理原始文本，因此需要先使用 TF-IDF 将英文简历文本转换为数值向量。

建议初始参数：

```python
from sklearn.feature_extraction.text import TfidfVectorizer

vectorizer = TfidfVectorizer(
    lowercase=True,
    stop_words="english",
    ngram_range=(1, 2),
    max_features=50000,
    min_df=2,
    max_df=0.9
)
```

说明：

| 参数 | 作用 |
| --- | --- |
| `lowercase=True` | 统一转小写 |
| `stop_words="english"` | 去除英文停用词 |
| `ngram_range=(1, 2)` | 使用 unigram 和 bigram |
| `max_features=50000` | 限制最大特征数量 |
| `min_df=2` | 过滤只出现 1 次的低频词 |
| `max_df=0.9` | 过滤过高频词 |

如果 TF-IDF 特征维度过高，XGBoost 训练速度较慢，可以考虑两种优化方式：

- 降低 `max_features`，例如设置为 20000 或 30000。
- 使用 `TruncatedSVD` 对 TF-IDF 特征降维，再输入 XGBoost。

### 6. FastText 数据格式

FastText 是轻量级文本分类模型，适合快速训练英文文本分类任务。FastText supervised 模式要求训练数据格式如下：

```text
__label__类别 文本内容
```

例如：

```text
__label__HR HR specialist with experience in recruiting employee relations onboarding benefits and training.
__label__INFORMATION-TECHNOLOGY Software engineer with Python SQL Linux database development and cloud service experience.
```

因此需要将原始数据转换为 FastText 训练文件：

```text
fasttext_train.txt
fasttext_test.txt
```

转换逻辑：

```python
def to_fasttext_line(text, label):
    text = clean_text(text)
    return f"__label__{label} {text}"
```

### 7. 基线模型选择

本项目基线阶段确定使用两个模型：

| 模型 | 输入特征 | 定位 |
| --- | --- | --- |
| XGBoost | TF-IDF 向量 | 机器学习强基线 |
| FastText | 原始英文文本 | 轻量级深度学习文本分类基线 |

两种模型的作用不同：

| 模型 | 优点 | 可能问题 |
| --- | --- | --- |
| XGBoost | 对结构化特征和非线性关系建模能力强，可作为机器学习强基线 | TF-IDF 特征维度高时训练较慢 |
| FastText | 训练速度快，适合文本分类，能利用词和 n-gram 信息 | 语义理解能力弱于 BERT |

推荐 baseline 对比方式：

```text
XGBoost(TF-IDF) vs FastText(supervised)
```

如果后续 BERT 模型上线，则最终形成三组对比：

```text
XGBoost baseline
FastText baseline
BERT model
```

## 十、模型评估方案

由于该任务是 24 类多分类，而且类别不均衡，因此不能只看 Accuracy。

建议评估指标：

| 指标 | 说明 |
| --- | --- |
| Accuracy | 整体预测正确率 |
| Macro Precision | 各类别精确率平均，关注小类别 |
| Macro Recall | 各类别召回率平均，关注小类别 |
| Macro F1 | 各类别 F1 平均，适合不均衡数据 |
| Weighted F1 | 按类别样本数加权后的 F1 |
| Confusion Matrix | 查看类别之间混淆情况 |

重点关注：

```text
Macro F1
```

因为 `BPO`、`AUTOMOBILE`、`AGRICULTURE` 等小类别样本较少，Accuracy 可能掩盖小类别识别效果差的问题。

## 十三、风险与问题

### 1. 数据量较小

虽然总数据量有 2484 条，但这是 24 分类任务，平均每类只有约 100 条样本，部分小类别不足 50 条。

解决思路：

- baseline 阶段先使用原始数据训练。
- 后续再使用数据增强方法扩充小类别。
- 使用分层抽样，避免小类别在训练集或测试集中缺失。

### 2. 类别不均衡

`BPO`、`AUTOMOBILE`、`AGRICULTURE` 等类别明显偏少。

解决思路：

- 评估时重点看 Macro F1。
- 模型训练时考虑 `class_weight="balanced"`。
- 后续对小类别进行数据增强。

### 3. 文本较长

简历平均长度超过 6000 字符，BERT 无法一次性读取完整文本。

解决思路：

- XGBoost 阶段使用 TF-IDF，不受 512 token 限制影响明显。
- FastText 阶段可直接处理较长文本，但仍建议清理异常空白和极端过长文本。
- BERT 阶段可采用截断、分块、摘要或 Longformer 等方案。

### 4. 数据来源偏单一

数据来自简历示例网站，可能存在模板化、领域偏差和真实简历差异。

解决思路：

- 在项目报告中说明数据来源和局限性。
- 后续可引入更多公开简历或岗位描述数据。

## 十四、后续升级方向

baseline 完成后，可以进一步升级：

### 1. XGBoost 与 FastText 对比分析

基线阶段结束后，首先对 XGBoost 和 FastText 进行对比：

| 对比维度 | XGBoost | FastText |
| --- | --- | --- |
| 输入 | TF-IDF 特征 | 原始文本 |
| 训练速度 | 中等，受特征维度影响 | 较快 |
| 可解释性 | 可通过特征重要性分析 | 相对较弱 |
| 文本语义理解 | 依赖词频和 n-gram | 利用词向量和 n-gram |
| 适合作用 | 机器学习强基线 | 轻量文本分类基线 |

### 2. Hierarchical BERT 长文本分类方案

本项目后续深度学习主线方案直接采用 Hierarchical BERT。

可使用英文预训练模型作为底层编码器，例如：

```text
bert-base-uncased
distilbert-base-uncased
roberta-base
```

输入：

```text
Resume_str
```

输出：

```text
24 类岗位类别
```

核心思想是将一份长简历切分为多个短文本片段，每个片段长度不超过 BERT 的最大输入长度，再分别输入 BERT 编码，最后将多个片段的语义表示聚合为整份简历的表示，用于岗位类别分类。

整体流程如下：

```text
Resume_str
-> 文本清洗
-> 按 token 窗口切分为多个 chunk
-> 每个 chunk 输入 BERT
-> 得到多个 chunk embedding
-> chunk 表示聚合
-> 分类层
-> 24 类岗位类别
```

示例：

```text
Resume_str
-> chunk_1: tokens 1-512
-> chunk_2: tokens 513-1024
-> chunk_3: tokens 1025-1536
-> chunk_4: tokens 1537-2048
```

每个 chunk 单独经过 BERT：

```text
chunk_1 -> BERT -> CLS_1
chunk_2 -> BERT -> CLS_2
chunk_3 -> BERT -> CLS_3
chunk_4 -> BERT -> CLS_4
```

再进行聚合：

```text
[CLS_1, CLS_2, CLS_3, CLS_4]
-> mean pooling / attention pooling
-> document embedding
-> classifier
```

最终输出：

```text
HR / INFORMATION-TECHNOLOGY / SALES / FINANCE / TEACHER / ...
```

#### 推荐初始参数

| 参数 | 建议值 | 说明 |
| --- | --- | --- |
| 预训练模型 | `bert-base-uncased` | 英文简历数据，先使用英文 BERT |
| 每个 chunk 长度 | 512 tokens | 与 BERT 最大输入长度一致 |
| 每份简历最大 chunk 数 | 4 到 8 个 | 控制显存和训练速度 |
| chunk 聚合方式 | mean pooling | 先用简单稳定方案 |
| 后续升级聚合方式 | attention pooling | 让模型学习不同片段的重要性 |
| 分类类别数 | 24 | 对应 24 个岗位类别 |

#### 为什么适合本项目

Hierarchical BERT 适合本项目的原因：

- 简历文本很长，直接截断会丢失信息。
- 简历天然具有层次结构，例如 Summary、Skills、Experience、Education。
- 每个片段可以独立编码，再汇总为整份简历表示。
- 相比 Longformer，Hierarchical BERT 更贴合当前 BERT 升级路线。
- 对当前 2484 条样本规模更稳，训练资源要求也更可控。

#### 最终模型路线

本项目模型路线调整为：

| 阶段 | 模型 | 作用 |
| --- | --- | --- |
| 基线模型 1 | XGBoost + TF-IDF | 建立机器学习基线 |
| 基线模型 2 | FastText supervised | 建立轻量文本分类基线 |
| 后续主模型 | Hierarchical BERT | 处理长简历文本，作为最终深度学习模型 |

### 3. 长文本处理

- 可尝试只保留 Summary、Skills、Experience、Education 等关键模块。
- 如果后续资源允许，可将 Longformer 作为扩展实验，与 Hierarchical BERT 进行对比。

## 十六、阶段计划

| 阶段 | 工作内容 | 产出 |
| --- | --- | --- |
| 第 1 阶段 | 数据分析和清洗 | 数据统计报告、清洗后文本 |
| 第 2 阶段 | baseline 模型训练 | XGBoost + FastText |
| 第 3 阶段 | 模型评估 | 分类报告、混淆矩阵、指标对比 |
| 第 4 阶段 | 数据增强 | 小类别扩充数据集 |
| 第 5 阶段 | BERT 升级 | Hierarchical BERT 长文本分类模型 |
| 第 6 阶段 | 项目部署 | Web 演示页面或 API 服务 |

## 十七、总结

本项目计划基于英文简历数据集构建岗位类别自动分类系统。当前阶段先使用原始英文数据训练两个 baseline 模型：`TF-IDF + XGBoost` 和 `FastText supervised`。前者作为机器学习强基线，后者作为轻量级深度学习文本分类基线。

这两个 baseline 模型将为后续 Hierarchical BERT 提供基础评估参考。最终主线模型采用 Hierarchical BERT，通过将长简历切分为多个 chunk，再聚合多个 chunk 的语义表示，解决普通 BERT 输入长度不足的问题。项目完成后，可以形成一个完整的简历分类流程，为招聘系统中的简历归类、岗位推荐和人才库管理提供基础能力。
